AP CSA Unit 3 – Big Idea 3

3.1 Abstraction and Program Design

  • Object-Oriented Design (OOD) → identify classes, attributes, methods.
  • Look for nouns to find class names.
  • UML: - private, + public.
  • Abstraction → hide unnecessary details.
  • Data Abstraction → separates properties from internal code.
  • Instance variable: unique per object. Class variable: shared by all (static).
  • Procedural Abstraction → named processes hide implementation.
  • Benefits: reuse, easier debugging, maintainability.

3.2 Impact of Program Design

  • System reliability: works consistently through testing.
  • Societal impact: can be helpful or harmful.
  • Legal & ethical issues → IP, fairness, autonomy.

3.3 Anatomy of a Class

  • Blueprint for creating objects.
  • Instance variables, constructors, methods.
  • Data Encapsulation → hide variables with private and use public accessors.

3.4 Constructors

  • Initialize object state.
  • Same name as class, no return type.
  • Default constructor if none written.

3.5 Methods

  • void vs non-void methods.
  • Accessors (getters) and mutators (setters).
  • toString returns a String representation automatically in print.
  • Parameters enable generalization.

3.6 Passing & Returning Objects

  • Objects passed by reference copy.
  • Mutable changes affect original.
  • Defensive copying prevents unwanted changes.
  • Returning objects → returns reference.

3.7 Static

  • Static variables shared by all objects.
  • Static methods called using class name (Math.random()).
  • Cannot access instance variables.
  • final → constants.

3.8 Scope

  • Class, method, block scope.
  • Shadowing hides instance variable.

3.9 this Keyword

  • Refers to current object.
  • Used when parameters shadow instance variables.
  • Not allowed in static methods.
