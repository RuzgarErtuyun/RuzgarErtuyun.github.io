Unit 3: Object Oriented Programming

3.1 Classes and Objects

  • A class is a blueprint for creating an object.
  • A class specifies the attributes (instance variables) and behavior (methods).
  • An object is an instance of a class.
  • Example: A "Dog" class and an actual dog object you create from it.
  • The "new" keyword creates an object (example: Dog d = new Dog()).

3.2 Using Objects

  • Attributes are the data inside an object (like color, height).
  • Methods define what the object can do.
  • Dot notation is used to access methods: objectName.methodName()
  • Objects can store data using instance variables.

3.3 Creating and using methods

  • Methods define behaviors of an object.
  • Return type tells what the method returns.
  • Void means the method does not return a value.
  • Parameters allow methods to accept input.
  • Method call: objectName.methodName(parameters)

3.4 Writing constructors

  • A constructor initializes the instance variables of an object.
  • Has the same name as the class.
  • Does NOT have a return type.
  • Constructors run automatically when you create an object.
  • If you write no constructor, Java provides a default one.

3.5 Documentation with comments and Javadoc

  • Comments explain code but don't affect how it runs.
  • // for single-line comments, /* */ for multi-line comments.
  • Javadoc comments use /** */ and generate official documentation.
  • Useful tags include @param and @return.

3.6 Overloading methods

  • Method overloading means having multiple methods with the same name but different parameters.
  • They must differ in parameter type, number, or order.
  • Return type alone cannot overload a method.

3.7 Class (Static) Variables and Methods

  • Static methods belong to the class, not objects.
  • Called using: ClassName.methodName().
  • Only one copy exists for the whole class.
  • main is static → only one per class.
  • Static methods in the same class can be called with class name or just method name.
  • Static methods cannot access instance variables unless given an object.
  • Static variables are shared by all objects and declared with the static keyword.
  • Useful for counters or shared values.
  • final keyword makes a variable constant (cannot change).

3.8 Scope and Access

  • Scope = where a variable can be accessed.
  • Class scope → instance variables available to all methods.
  • Method scope → local variables exist only in the method.
  • Block scope → variables inside loops or if-statements.
  • Local variables cannot be public or private.
  • Parameters are local variables.
  • Local variable names override instance variables with the same name.
  • this keyword is used to refer to instance variables when shadowed.
  • Variables cannot be used outside their scope.
